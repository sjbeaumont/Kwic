# Kwic Contribution Guide

Thank you for considering contributing to Kwic! Contributors help improve the tool for everyone.

### What Contributions are we Looking For?

Kwic welcomes any contribution that will move the project foward; this doesn't just have to be code. You can request new features, submit bug reports, or, obviously, write/change code that could be incorporated into Kwic.

### Responsibilities

* **Ensure compatability with Linux.** This tool is designed for Linux, not Windows, or Mac... yet.

* **Don't add commands unless absolutely needed.** Kwic aims to be a small program, we don't want new commands simply because they'll do cool stuff.

* **Be respectful.** If someone's code isn't looking right, point it out nicely, and offer a suggestion on how to fix it.

<br>

> *Working on your first Pull Request? Me too 😅. Check out how to get started [here.](http://firsttimersonly.com)

### How to Contribute

For something more than just a couple line fix:

1. Make your own fork of the code
2. Do the changes in your fork
3. Once satified with the change, ensuring it works, if you think the project could use it, then make a PR (view [PR Template](pr-template.md)), and all done!

<br>

However, if it's just a small fix such as:

* Spelling / grammar

* Formatting / comments

* Moving source files from one directory or another

You can just make a [PR](pr-template.md) and write it on there.
